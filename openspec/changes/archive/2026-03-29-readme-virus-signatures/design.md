## 背景

儲存庫目前無任何說明文件。使用者與貢獻者無法從 GitHub 頁面快速了解引擎功能、建置方式或特徵資料庫格式。病毒特徵 hash 的來源需要公開且可引用的資料，以確保教育用途的合法性與可信度。

## 目標 / 非目標

**目標：**
- 建立完整的 `README.md`，提供足夠的資訊讓新使用者從零開始使用本引擎
- 在病毒特徵 hash 參考表中，列出來自公開已記錄來源的 SHA256 值，並附上出處引用
- 以 YouTube 教學影片連結作為學習延伸資源
- 說明特徵 YAML 格式，讓使用者能自行擴充資料庫

**非目標：**
- 修改任何 Go 原始碼或設定檔
- 提供完整的 API 文件（由 godoc 負責）
- 包含真實惡意軟體的下載連結

## 決策

### D1：README 語言使用繁體中文
**決定**：README 主要內容以繁體中文撰寫，技術術語（CLI 指令、程式碼區塊、檔案路徑）保持英文。
**原因**：與本專案所有 OpenSpec 文件語言一致，降低繁體中文使用者的閱讀門檻。

### D2：病毒 hash 僅使用公開記錄來源
**決定**：僅收錄來自 eicar.org、CISA、US-CERT 等官方或學術機構公開記錄的 SHA256 hash。
**原因**：確保教育用途合法性，避免引用可疑或未驗證的惡意軟體樣本。

### D3：README 結構
README 章節順序：
1. 專案標題與一行概述
2. 架構圖（ASCII）
3. 快速開始（環境需求、建置、執行）
4. 特徵 YAML 格式說明
5. 病毒特徵 hash 參考表
6. 結束碼說明
7. 參考資源

**原因**：將最常用的資訊（快速開始）放在最前面，hash 參考表置於中後段供查閱使用。

## 風險 / 取捨

- **hash 資料老化** → 公開惡意軟體樣本的 hash 為靜態值，不會過期，但需標示記錄日期
- **語言混合** → 中文說明混合英文技術術語，可能影響非中文讀者；但程式碼區塊全程保持英文可降低此影響
